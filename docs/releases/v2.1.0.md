# Bloom v2.1.0

**Release date:** July 14 2026

## Overview

Bloom v2.1.0 moves the application from primarily browser-based prototype storage toward authenticated full-stack persistence.

The release connects tasks, focus tasks, routines, routine steps, and progress snapshots to the FastAPI backend while preserving separate guided-demo behaviour.

## Added

- Authenticated Home-task persistence
- Task emoji persistence
- Authenticated Focus-task persistence
- Selected Focus-task persistence
- Authenticated routine and routine-step persistence
- Starter routines for authenticated users
- Cross-page routine synchronisation
- Backend progress snapshots
- Loading and error states for persisted data
- Active-page persistence after refresh

## Fixed

- Routines created on the Routines page not appearing on Home
- Starter routines not persisting for authenticated users
- Demo/local routine IDs reaching backend routes
- Focus selected-task state not clearing after deletion
- Progress routine totals not updating correctly
- Duplicate authentication checks
- Progress wording and milestone copy
- Broken Moments “View all wins” interaction
- Demo focus history persisting between sessions
- Demo tasks, routines, focus tasks, reflections, and progress leaking between demo sessions
- Refresh returning authenticated users to Home

## Technical improvements

- Separated authenticated backend state from demo/local state
- Added event-driven synchronisation between Routines and Home
- Replaced derived Progress state effects with memoised calculations
- Removed dead imports and unused state
- Improved cleanup of local and session storage
- Preserved backend-issued numeric IDs for authenticated resources

## Verification

- End-to-end authenticated account flow passed
- End-to-end demo flow passed
- Refresh persistence verified
- Logout and login persistence verified
- Routine, task, Focus, and Progress persistence verified
- Targeted ESLint checks passed
- Production frontend build passed

## Known limitations

- Backend starter-routine creation is sequential and does not currently use rollback
- Backend starter routines are not yet deduplicated by name
- Backend routine completion does not yet support configurable daily reset policies
- Focus minutes and reflections are not yet persisted across devices
- “View all wins” is planned for v2.1.1
- Additional themes, languages, accessibility controls, and keyboard shortcuts are planned for later releases
