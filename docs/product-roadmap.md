# Product Roadmap

This document contains Bloom’s current product roadmap and future development direction.

## Current Phase

Bloom is currently moving into the **v2.1.0 authentication phase**.

The main focus is stabilising account-based app usage while keeping demo mode separate from real account data.

## v2.1.0 Focus

Planned work:

- Stable login flow
- Stable account creation flow
- Logout flow
- Restore current user on refresh
- Protect app pages from unauthenticated access
- Keep demo mode separate from real account data
- Save selected avatar per user
- Improve loading states
- Improve error states
- Final manual testing

## Backend and Persistence Roadmap

Future backend work will include:

- Backend-connected feedback storage
- Database persistence for routines
- Database persistence for tasks
- Database persistence for focus items
- Database persistence for moments
- Saved profile settings
- Saved accessibility preferences
- Saved avatar per user

## Future Product Features

Planned future features:

- Onboarding flow
- Mood check-in at app open
- Short version of routines
- Oasis calm reset space
- Low demand mode
- Exportable progress CSV
- More guided demo options
- Optional reminders
- Future full-stack deployment

## Beta Direction

Bloom Personal is being prepared for public beta feedback.

The beta is focused on:

- Calm user experience
- Routine flow clarity
- Accessibility and readability
- Demo mode usability
- Feedback from neurodivergent users
- Small, focused tester group

## Long-Term Direction

Bloom may later grow into a fuller routine, focus, and reflection support app with:

- Account-based saved routines
- Progress history
- More profile customisation
- Optional export features
- Accessibility-first settings
- Possible education/guardian version later, only after privacy and safeguarding planning
