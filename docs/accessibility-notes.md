# Accessibility Notes

This document explains the accessibility and neurodivergent-friendly design direction for Bloom.

## Design Goal

Bloom is designed to feel calm, clear, and supportive.

The app avoids aggressive productivity language and focuses on gentle task sequencing, visual clarity, and low-pressure interaction patterns.

## Current Accessibility Features

Current features include:

- Light and dark mode
- Text size controls
- OpenDyslexic font toggle
- Reduce motion setting
- Calm visual design
- Responsive mobile layouts
- Soft copywriting
- Clear task/routine structure
- Public Accessibility page

## Neurodivergent-Friendly Design Choices

Bloom prioritises:

- Low-pressure wording
- Clear visual hierarchy
- Predictable page structure
- Gentle task sequencing
- Reduced overwhelm
- Soft recovery support
- Calm colour direction
- Avoiding shame-based productivity language
- Making missed days feel recoverable

## UI Notes

Current UI direction includes:

- Calm routine cards
- Focus session support
- Progress and reflection pages
- Moments dashboard
- Readability controls
- Protected app layout with sidebar/header
- Public page layout separated from protected app layout

## Future Accessibility Improvements

Future accessibility work may include:

- More keyboard navigation checks
- Better focus states
- More screen reader testing
- Reduced-motion animation review
- More onboarding clarity
- More low-demand flow options
- Saved accessibility preferences per user
