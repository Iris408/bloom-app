# Demo Mode

This document explains Bloom’s demo mode direction.

## Demo Mode Purpose

Demo mode lets users experience Bloom without needing an account.

It is useful for:

- First-time users
- Beta testers
- Recruiters
- People who want to explore before signing up
- Users who may feel overwhelmed by account creation

## Current Demo Direction

Demo mode is launched from the public Overview page.

It allows users to explore Bloom’s core app experience, including routines, focus, progress, and moments.

## Demo Goals

Demo mode is made to feel:

- Calm and Clear
- Low-pressure
- Lightweight
- Easy to exit
- Separate from real account data

## Demo Boundaries

Demo mode should stay separate from real user data.

Future account-based data will be separate from demo routines, demo tasks, or demo moments.

## Current Demo Options

Future guided demos may include:

### Simple Day

A lightweight demo with:

- One routine
- One focus block
- A few gentle daily steps

### Neurodivergent-Friendly Day

A calm demo with:

- Low-pressure wording
- Recovery support
- Sensory-friendly pacing
- Gentle focus flow

### Full App Preview

A broader demo with:

- Routines
- Focus
- Progress
- Moments
- Settings
- Accessibility controls

### Current Demo Features

- Demo setup choice before starting
- Preloaded routine options
- Small completion celebration

## Future Demo Improvements

Possible improvements:

- Demo moment after completing a task/routine
- Clear reset demo option
- Better separation between demo and real account state
